HTML Structure - top

This theme comes with two liquid layouts: a wide layout and a boxed layout.

All of the major sections and essential elements in the HTML documents are well commented and named according to their purpose.

CSS Files and Structure - top

I'm using several CSS files in this theme. Most of them were imported at the beginning of the master stylesheet, that is, "style.css".

The master stylsheet, that is, "style.css", is located in the root directory. It contains the stylings for the major elements of the website.

The other stylesheets are located in the "css" directory. These are:

  • structural stylesheets: "html5-reset.css";
  • a shortcode stylesheet: "shortcodes.css";
  • color stylesheets (located in the "colors" subdirectory);
  • plugin-specific stylesheets: "flexslider.css", "jquery.fancybox.css", "mediaelementplayer.css", "polyglot-language-switcher.css";
  • a browser-specific stylesheet: "ie.css" which contains fixes for the Internet Explorer browser.

All the CSS stylesheets that were created by me are prefaced by a table of contents and are separated into sections using comments. This allows an easy identification of their content.

Applying the Styles Available in the Style Switcher - top

You can permanently apply the styles available in the style switcher by making a few modifications.

Changing the Layout Style

By default, both the HTML and PHP template versions use the wide layout version.

The only difference between the wide and boxed layout versions is the "style.css" file.

To transform the website from wide to boxed, overwrite the "style.css" file from the HTML or PHP folder with the "style.css" file located in the "boxed" folder.

Changing the Theme Color

In the head of the HTML pages, you will find the link to the color stylesheet, that is, <link href="css/colors/orange.css" type="text/css" rel="stylesheet" id="color-style">.

Change the name of the default color stylesheet, that is, orange.css, to the name of the stylesheet corresponding to your desired color. The color stylesheets are located at the following path: css/colors.

Changing the Background Color and/or Pattern

Open the "style.css" stylesheet and go to this rule:

body {
	font-family: 'Open Sans', sans-serif;
    font-size: 12px;
    line-height: 1.5em; /* 18px */
    color: #383838;
    background-color: #fff;
    background-image: none;
    background-repeat: repeat;
    background-position: left top;
    /* Fix for webkit rendering */
    -webkit-font-smoothing: antialiased;
    -webkit-text-size-adjust: 100%;
}

Change the expressions highlighted in red to your desired color and/or the URL of your desired pattern respectively. The background patterns are located at the following path: images/background-patterns.

For example, you could have:

background-color: #fcfcfc;
background-image: url(images/background-patterns/wide/subtle-dots.png);

Translating the Twitter Widget

To translate the Twitter Widget relative time to different languages, you must follow these steps:

Search in all pages for: <div class="tweet"></div> and use the following HTML5 data attributes to provide a new relative time translation:

  1. data-just-now-text
  2. data-seconds_ago-text
  3. data-a-minutes-ago-text
  4. data-minutes-ago-text
  5. data-a-hours-ago-text
  6. data-hours-ago-text
  7. data-a-day-ago-text
  8. data-days-ago-text

Example:

<div class="tweet"
        data-just-now-text="just now"
        data-seconds-ago-text="about %d seconds ago"
        data-a-minutes-ago-text="about a minute ago"
        data-minutes-ago-text="about %d minutes ago"
        data-a-hours-ago-text="about an hour ago"
        data-hours-ago-text="about %d hours ago"
        data-a-day-ago-text="about a day ago"
        data-days-ago-text="about %d days ago"></div>

ATENTION!!!

  1. The value %d must not be removed from the translation text!
  2. The data attributes values must be properly escaped. For the HTML version, you can use the following website for HTML escaping: http://www.htmlescape.net/htmlescape_tool.html. For the PHP version, you can do this by using the htmlspecialchars method.

Example of escaping the data value using PHP:

<div class="tweet"
        data-just-now-text="<?php echo htmlspecialchars('just now'); ?>"
        data-seconds-ago-text="<?php echo htmlspecialchars('about %d seconds ago'); ?>"
        data-a-minutes-ago-text="<?php echo htmlspecialchars('about a minute ago'); ?>"
        data-minutes-ago-text="<?php echo htmlspecialchars('about %d minutes ago'); ?>"
        data-a-hours-ago-text="<?php echo htmlspecialchars('about an hour ago'); ?>"
        data-hours-ago-text="<?php echo htmlspecialchars('about %d hours ago'); ?>"
        data-a-day-ago-text="<?php echo htmlspecialchars('about a day ago'); ?>"
        data-days-ago-text="<?php echo htmlspecialchars('about %d days ago'); ?>"></div>

Elements Customization - top

The following elements can be customized through the HTML5 attributes:

Tabs

  • data-active - specify the active tab. The value should be a number between 1 and n, where n is the number of tabs
  • data-disabled - specify the disabled tabs. The value should be a number between 1 and n, where n is the number of tabs or a list of comma-separated numbers
  • data-height-style - controls the height of the tabs widget and of each panel. Possible values:
    1. auto - all panels will be set to the height of the tallest panel
    2. fill - expand to the available height based on the tabs' parent height
    3. content (default) - each panel will be only as tall as its content
Example
<div class="tabs" data-active="2" data-disabled="3,4">
    <ul class="nav clearfix">
        <li><a href="#tab-1">Tab 1</a></li>
        <li><a href="#tab-2">Tab 2</a></li>
        <li><a href="#tab-3">Tab 3</a></li>
        <li><a href="#tab-4">Tab 4</a></li>
    </ul>
    <div id="tab-1" class="tab"><p>Content 1</p></div>
    <div id="tab-2" class="tab"><p>Content 2</p></div>
    <div id="tab-3" class="tab"><p>Content 3</p></div>
    <div id="tab-4" class="tab"><p>Content 4</p></div>
</div>

Toggles

  • data-active - specify an active section. The value should be true of false
  • data-disabled - specify is the section is disabled. The value should be true of false
  • data-height-style - controls the height of each panel. Possible values:
    1. auto - all panels will be set to the height of the tallest panel
    2. fill - expand to the available height based on the tabs' parent height
    3. content (default) - each panel will be only as tall as its content
Example
<div class="toggle">
    <span class="toggle-title">Section 1</span>
    <div class="toggle-inner"><p>Content 1</p></div>
</div>
<div data-active="true" class="toggle">
    <span class="toggle-title">Section 2</span>
    <div class="toggle-inner"><p>Content 2</p></div>
</div>
<div data-disabled="true" class="toggle">
    <span class="toggle-title">Section 3</span>
    <div class="toggle-inner"><p>Content 3</p></div>
</div>
<div data-active="true" class="toggle">
    <span class="toggle-title">Section 4</span>
    <div class="toggle-inner"><p>Content 4</p></div>
</div>

Accordion

  • data-active - specify the opened section. The value should be a number between 1 and n, where n is the number of sections
  • data-disabled - specify is the all accordion is disabled. The value should be true of false
  • data-height-style - controls the height of each panel. Possible values:
    1. auto - all panels will be set to the height of the tallest panel
    2. fill - expand to the available height based on the tabs' parent height
    3. content (default) - each panel will be only as tall as its content
Example
<div class="accordion" data-active="2">
    <div>
        <span class="accordion-title">Title 1</span>
        <div class="accordion-inner"><p>Content 1</p></div>
    </div>
    <div>
        <span class="accordion-title">Title 2</span>
        <div class="accordion-inner"><p>Content 2</p></div>
    </div>
    <div>
        <span class="accordion-title">Title 3</span>
        <div class="accordion-inner"><p>Content 3</p></div>
    </div>
    <div>
        <span class="accordion-title">Title 4</span>
        <div class="accordion-inner"><p>Content 4</p></div>
    </div>
</div>

Contact & Administration Dashboard - top

The Administration Dashboard allows you to manage the following features:

  1. Website Search
  2. Sending of Newsletters to the Subscribers

The settings regarding the above features are stored in a database. The supported databases are:

  1. SQLite (v2.x and v3.x)
  2. MySQL 5.x

By default, the type of database used by the Administration Dashboard is SQLite (its version doesn't matter). I chose this database because you don't have to do anything to configure it, so it is very simple to use. If your web hosting package doesn't support this database, then you can choose to switch to MySQL. (Go to the MySQL configuration.)

To access the Administration Dashboard and use the Contact Form, you must replace the "abc@xyz.com" email address in "settings.php" with your own email address. Please note that the domain of your email address must be the same as the domain of your website. In other words, don't use yahoo, google, msn, etc. emails.

This email address will be used in both the contact page and the newsletter page.

  1. Unlock Access to the Admin Page

    The administration page is private and, first, you must unlock it by generating a private key.

    To do this, access the "admin.php" page (for example, "www.yoursite.com/admin.php"), click the "Unlock" button and verify your email. Here you will receive a link which will allow you to access the admin page.

  2. Index Your Website for Search

    To have a functional search mechanism, you must index your website. This can be done by clicking the "Rebuild search indexes" link on the right side of the admin page.

    The indexing process must be run each time you modify, add or delete a page of your website. By default, the folder where the indexes are kept is located in your "tmp" folder on your web server.

    Note: During the website indexing process, don't click the "Rebuild search indexes" link until the success notification box is displayed.

    Sometimes, after you click the "Rebuild search indexes" link, it is possible to get an error like

    Fatal error: Call to a member function removeChild() on a non-object …

    This happens when the HTML files that are going to be indexed aren’t properly formatted (for example, all tags weren’t closed).

    To fix this error, in the "Zend/Search/Lucene/Document/Html.php" file, go to line 117 and comment it by adding a double slash (//) at its beginning:

    //$head->parentNode->removeChild($head);

    After you do this, try to rebuild the search indexes again.

  3. Configure your MySQL Database

    This step must be performed only if you choose to use the MySQL database instead of the SQLite database.

    The type of database used by the Administration Dashboard if given by the value of the "$db_type" property in the "settings.php" file. This property accepts only two values: mysql for the MySQL database and sqlite for the SQLite database. For any value different from these two, an error message will be displayed and the Administration Dashboard will not work.

    To use the MySQL database instead of the SQLite database, you must change the values of the following parameters in the "settings.php" file as follows:

    $db_type='mysql';
$db_host='database_hostname';
$db_name='database_name';
$db_user='database_user';
$db_pass='database_user_password';

    Replace the values marked with red with your own values as follows:

    $db_host – must reflect the name of the server where your database is located. (This is provided by your web host.)

    $db_name – the database name (You must create an empty database.)

    $db_user – the name of the user associated with the database (You must associate a user with your database.)

    $db_pass – the password for the database user.

Explanation of the Admin Configuration Parameters

  • The name of the folder where the website is located – This is the name of the folder on the server where your website is located. It can have one of the following values (but not always): public_html, htdocs, httpdocs.

  • Path to the folder that contains the website indexes – This is the folder location where the indexes are kept. I recommend not to change this property unless it is necessary.

  • Search results items per page – This is the number of result items displayed per page.

  • Search description length – When the search results are displayed, each result item provides, besides the page title and page link, a short description of that page. This parameter sets the length of this description.

  • List of dynamic pages (Enter one URL per line.) – By default, the indexing mechanism will index only the static pages of your website (that is, the HTML pages). If, beside static pages, you also have dynamic pages (that is, pages written in PHP, Perl, Python or CGI scripts), you must provide the links to these pages, one per line. (After each URL, press enter.)

FAQ

  • I didn't receive the email with the private key. What should I do?

    Go to your web hosting and open the "settings.php" file. It's there that your private key is kept. Copy the private key without the single quotes and paste it like in the following link: "www.yoursite.com/admin.php?pk=the-private-key".

  • I lost my private key. How can I recover it?

    Go to your web hosting and open the "settings.php" file. It's there that your private key is kept. Copy the private key without the single quotes and paste it like in the following link: "www.yoursite.com/admin.php?pk=the-private-key".

  • I want to generate another private key. What should I do?

    In the "settings.php" file, delete all the content bellow the dividing line. Follow the step no. 2. An email with a new private key will be sent to you.

  • I want to change the email subject of the newsletter. What should I do?

    Go to "newsletter.php" and modify the content of the "$newsletter_email_subject" variable as follows: $newsletter_email_subject = 'your custom subject';

    Note: Your subject message must be enclosed in apostrophes.

ReCaptcha Activation

By default, the ReCaptcha code is disabled. If you want to enable it, in the settings.php file, you must set the value of the ENABLE_CAPTCHA constant to true like this:

define('ENABLE_CAPTCHA', true);

After that, you must configure the ReCaptcha code generator by providing the Private and the Public keys following the instructions below.

ReCaptcha Configuration

  1. Open the http://www.google.com/recaptcha website and click on My Account link



  2. Create ReCaptcha keys for your website: (the Private and the Public Keys)




  3. Copy the Private Key and the Public Key in the settings.php file

If you are using the HTTPS protocol for your website, then, in the setings.php file, you must set the value of the USE_SSL constant to true like this:

define('USE_SSL', true);

F) PHP Template Version - top

Every page includes a common header and footer located in the "includes" folder. This helps the user apply the changes regarding these parts in a single place.

Defining the Menu

In the "menu.php" file located in the "includes" folder, you will find an XML that defines the menu structure.

The XML contains only two types of tags:

  1. menu – used to wrap the entire menu inside it
  2. menu_item – used to describe a menu item
    This tag contains two attributes:
    • href – the page url
    • label – the menu item label

    3. This tag can also contain children.

Predefined PHP Functions

The menu.php contains two main functions:

  1. print_main_menu() – print the main navigation menu (this function is called in header.php)
  2. print_sitemap() – print the site map based on the menu structure (this function is called in sitemap.php)
